|-----------|---------|------------|
| (1) | 3つの取得メソッドの使い分け | Retrieve / Get / List の比較 |

## 学習ポイント

### (1) 3つの取得メソッド

| メソッド | スコープ指定 | 主な用途 |
|---------|------------|--------|
| `retrieve()` | 必要（完全一致） | プロンプト構築、類似検索 |
| `get()` | 不要（name指定） | 特定の記憶のピンポイント参照 |
| `list()` | 不要 | 全体把握（デバッグ用途） |

### (2) スコープの完全一致制約

- `Retrieve` のスコープ指定は **完全一致** が必要
- 複合キーの部分一致では取得できない
- **アンチパターン**: スコープで細かく分けすぎると横断検索ができない
- **回避策**: 横断検索したい属性は **メタデータ** に持たせる
  - スコープ = 越えられない壁（広めに設定）
  - メタデータ = 柔軟なフィルタ（細かい絞り込み用）

### (3) 2種類のフィルタリング

#### A. メタデータフィルタ（`filter_groups`）

- **形式**: DNF（論理和標準形: AND の OR）
- **特徴**: 完全一致のみ（部分一致・大小比較は不可）
- **対象**: 開発者が `generate()` / `create()` 時に付与したメタデータ

```python
config={
    "filter_groups": [{
        "filters": [
            {"key": "category", "value": {"string_value": "learning"}}
        ]
    }]
}
```

#### B. システムフィールドフィルタ（`filter`）

- **形式**: EBNF 構文の文字列
- **対象と演算子**:
  - `fact`: `=~` で正規表現部分一致 → `fact=~".*Python.*"`
  - `create_time` / `update_time`: `>=`, `<=` で日時範囲 → `create_time>="2026-01-01T00:00:00Z"`
  - `topics`: `:` (HAS演算子) → `topics.managed_memory_topic: USER_PREFERENCES`

#### C. 複合フィルタ

- `filter_groups` と `filter` を同時に指定可能
- 両方の条件を満たすメモリだけが返る

### (4) セマンティック検索

- `similarity_search_params` で有効化
- **search_query**: 検索クエリ（意味的な類似度で比較）
- **top_k**: 返す件数
- キーワード一致ではなく「意味的な類似度」に基づく検索
- ユークリッド距離が最も短いものから順にソートされて返される
- `distance` が小さいほど類似度が高い
